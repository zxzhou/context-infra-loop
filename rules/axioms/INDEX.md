# Axioms Index

> **Axioms** are the user's core cognitive filters: deep philosophical principles distilled from personal experience.
>
> Unlike Skills, which live at the execution layer ("how to do it"), Axioms live at the thinking layer ("why"). They are used to spark deeper thinking and provide alternative perspectives.

---

## How to Use

### Quick Scan (Index)

**When to read the index**:
- At the start of a task involving strategic decisions or architecture design
- When you want to understand the user's cognitive baseline
- When you need to find the axiom category relevant to the current task

### Deep Reading (Detailed Files)

**When to read the detailed files**:
- When the task touches the user's core values or methodology
- When you need to understand an axiom's deeper reasoning and applicability tests
- When the user explicitly asks you to reference their personal philosophy
- When doing reflective work, such as distilling axioms or deep retrospectives

### Core Axiom Clusters

These axioms form an organic whole and are often invoked together:

- **AI Management Triangle**: A02 (multiplier) + A03 (IC to manager) + A04 (reliability) - the baseline mindset for all AI collaboration
- **Cognitive Asset Cluster**: T05 (cognition is an asset) + A02 + A03 - value repositioning after the cost of code collapses
- **Verification Loop**: V02 (verifiability) + T07 (isolate-process-verify) + V04 (temporal grounding) - building trustworthy AI systems

### Active Retrieval Triggers

When the user mentions these keywords, proactively consult the related axioms:

- "manage AI", "delegate", "delegation" -> A03, A04
- "framework choice", "technical selection", "lock-in" -> A06, T08
- "verification", "trust", "reliability" -> V series, A04
- "priority", "efficiency", "bottleneck" -> M03, X03
- "context", "memory", "documentation" -> A05, T03
